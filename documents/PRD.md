## 1. 产品概述

### 1.1 产品愿景

LuminaLink摄像头实时解说系统旨在帮助视障用户通过摄像头对准屏幕（电影院或家庭影院），获得实时AI生成的场景解说。系统在检测到角色对话时自动静音，在对话间隙提供场景描述，为视障用户提供无障碍的观影体验。

### 1.2 产品定位

- **产品类型**: 桌面GUI应用（基于Python + Tkinter）
- **目标平台**: macOS（优先），未来支持Windows/Linux
- **核心价值**: 实时、智能、无障碍的电影解说服务

### 1.3 背景与动机

**现状问题**:
- 电影院和流媒体平台的音频描述（Audio Description）服务覆盖率低
- 现有AD服务需要提前制作，无法应用于所有内容
- 视障人士在家庭观影时缺少实时辅助工具

**解决方案**:
- 利用AI视觉理解技术（GPT-4 Vision）实时分析画面
- 通过摄像头捕获屏幕内容，无需修改播放设备
- 智能检测对话，避免与原音轨冲突
- 提供简单易用的GUI界面，支持无障碍操作

## 2. 目标用户

### 2.1 主要用户画像

**用户A: 盲人电影爱好者**
- **年龄**: 25-65岁
- **需求**: 在家中或电影院观看电影时获得场景描述
- **痛点**: 现有电影大部分没有音频描述轨道
- **期望**: 能够独立使用设备，快速启动解说，画面清晰时解说准确

**用户B: 低视力用户**
- **年龄**: 40-70岁
- **需求**: 看得到画面但无法辨识细节，需要补充说明
- **痛点**: 无法识别远处人物、动作细节
- **期望**: 解说与画面同步，不遮挡可见部分

### 2.2 次要用户

- **家庭成员/护工**: 帮助视障用户设置和使用系统
- **无障碍技术倡导者**: 推广和测试系统
- **研究人员**: 研究AI辅助视障技术

## 4. 用户旅程

### 4.1 首次使用流程

```
1. 用户下载并安装LuminaLink
   ├─ 安装Python依赖
   └─ 配置OpenAI API密钥

2. 用户启动应用
   ├─ 运行命令: python -m src.main camera
   └─ GUI窗口打开，状态显示"Ready"

3. 用户选择摄像头（如有多个）
   └─ 从下拉菜单选择"FaceTime HD Camera"

4. 用户点击"开始"按钮
   ├─ 摄像头画面出现在预览窗口
   └─ 状态变为"Running"

5. 用户将摄像头对准电影屏幕
   └─ 调整角度确保画面清晰

6. 系统开始分析和解说
   ├─ 每5-10秒分析一次画面
   ├─ 在静音期生成并播放解说
   └─ 字幕显示在画面底部

7. 用户观影过程中
   ├─ 听到解说时：继续观影
   ├─ 需要暂停时：按空格键或点击"暂停"
   └─ 结束观影时：按ESC或点击"停止"
```

### 4.2 日常使用流程

```
1. 启动应用: python -m src.main camera
2. 点击"开始"（上次设置已保存）
3. 对准屏幕开始观影
4. 观影结束后点击"停止"
```

### 4.3 高级使用流程（角色识别）

```
1. 准备角色配置文件
   └─ 创建 characters.json 包含角色信息

2. 启动应用并加载配置
   └─ python -m src.main camera --characters characters.json

3. 系统识别角色并在解说中使用真实姓名
   └─ 例如: "Tony Stark走进房间" 而非 "一个男人走进房间"
```

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| **端到端延迟** | <500ms（理想<300ms） | 从摄像头捕获到TTS播放开始 |
| **视频捕获帧率** | 30 FPS | 摄像头实际捕获速率 |
| **场景分析频率** | 5-10 FPS | AI分析调用频率 |
| **GUI刷新率** | ≥15 FPS | 视频预览窗口刷新 |
| **内存占用** | <500 MB | 运行时内存峰值 |
| **CPU占用** | <50%（空闲<10%） | 平均CPU使用率 |
| **启动时间** | <5秒 | 从运行命令到GUI显示 |

### 5.2 无障碍需求

| 需求 | 实现方式 |
|------|---------|
| **纯键盘操作** | 所有功能可通过键盘访问（Tab导航，快捷键） |
| **屏幕阅读器兼容** | 与macOS VoiceOver完全兼容 |
| **状态语音提示** | 所有状态变化（开始/暂停/停止/错误）通过TTS播报 |
| **高对比度模式** | 支持系统高对比度主题 |
| **可调字体大小** | 界面文字大小可配置 |
| **音频反馈** | 操作成功/失败有提示音 |

### 5.3 可靠性需求

| 场景 | 预期行为 |
|------|---------|
| **摄像头断开** | 显示错误提示，不崩溃，支持重新连接 |
| **API失败** | 显示警告，跳过当前帧，继续分析后续帧 |
| **网络中断** | 显示连接错误，提示检查网络，支持重试 |
| **麦克风不可用** | 显示警告，禁用对话检测，继续解说 |
| **系统资源不足** | 降低分析频率，显示性能警告 |
| **长时间运行** | 连续运行30分钟以上无内存泄漏 |

### 5.4 兼容性需求

| 平台 | 版本要求 | 优先级 |
|------|---------|--------|
| **macOS** | 12.0+ (Monterey及以上) | P0 |
| **Windows** | 10/11 | P2 |
| **Linux** | Ubuntu 20.04+ | P2 |
| **Python** | 3.8+ | P0 |

## 7. 数据需求

### 7.1 输入数据

| 数据类型 | 来源 | 格式 | 说明 |
|---------|------|------|------|
| **视频流** | 摄像头设备 | BGR图像（NumPy数组） | 实时捕获，30 FPS |
| **音频流** | 麦克风 | PCM音频（22050 Hz） | 用于对话检测 |
| **角色配置** | JSON文件（可选） | `{"characters": [...]}` | 人物信息 |
| **用户设置** | JSON文件 | `~/.luminalink/settings.json` | 用户偏好 |

### 7.2 输出数据

| 数据类型 | 输出目标 | 格式 | 说明 |
|---------|---------|------|------|
| **解说音频** | 扬声器 | MP3/WAV（EdgeTTS） | 实时播放 |
| **解说文本** | GUI字幕 | UTF-8字符串 | 显示在画面上 |
| **日志记录** | 文本文件（可选） | 时间戳+文本 | 用于回顾 |

### 7.3 配置数据

**角色配置示例** (`characters.json`):
```json
{
  "characters": [
    {
      "name": "Tony Stark",
      "aliases": ["Iron Man", "Tony"],
      "description": "The main character, genius billionaire"
    }
  ]
}
```

**用户设置示例** (`settings.json`):
```json
{
  "camera_index": 0,
  "tts_voice": "en-US-AriaNeural",
  "tts_speed": 1.0,
  "narration_interval": 5,
  "show_subtitles": true,
  "audio_threshold": -40
}
```

## 8. 技术约束

### 8.1 外部依赖

| 依赖 | 用途 | 约束 |
|------|------|------|
| **OpenAI API** | GPT-4 Vision场景分析 | 需要API密钥，有使用成本 |
| **EdgeTTS** | 文本转语音 | 需要网络连接 |
| **摄像头设备** | 视频输入 | 必须有可用摄像头 |
| **麦克风设备** | 音频输入 | 建议有麦克风（可选） |

### 8.2 性能约束

- **AI推理延迟**: GPT-4V API调用通常150-300ms，是延迟主要来源
- **网络带宽**: 每次API调用上传~100KB图像数据
- **本地计算**: 视频处理和音频分析占用CPU资源

## 10. 风险与假设

### 10.1 风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| OpenAI API延迟过高 | 中 | 高 | 添加本地模型支持（LLaVA） |
| 音频检测不准确 | 高 | 中 | 提供手动暂停；可调阈值 |
| 摄像头兼容性问题 | 低 | 高 | 启动时检测；提供设备列表 |
| 成本过高（API费用） | 中 | 中 | 提供本地模型选项；控制调用频率 |
| 无障碍不达标 | 低 | 高 | 早期邀请视障用户测试 |

### 10.2 假设

- ✓ 用户有可用的摄像头设备
- ✓ 用户有稳定的网络连接（用于API调用）
- ✓ 用户有OpenAI API密钥或愿意注册
- ✓ 用户的设备性能足够（不是过于老旧的硬件）
- ✓ 摄像头能够清晰拍摄到屏幕画面

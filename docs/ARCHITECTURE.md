# System Architecture (Phase 1: Vision-to-Text)

## 1. Overall Data Flow

Input: Video File/Stream → Decode → Frame/Audio Distribution → Visual Understanding & Dialogue Detection → Narration Generation → Time Alignment → Output.

## 2. Core Design Principles

- Single Responsibility: Each module does one thing (Detection/Recognition/Generation/Alignment/Output).
- Pluggable: Model implementations are replaceable (Different inference backends/training versions).
- Time Priority: All intermediate artifacts must carry a unified timestamp (ms).
- Streaming Processing: Use queues and sliding windows to control latency and throughput.

## 3. Module Partitioning (Suggested Interfaces)

- `InputAdapter`: Outputs `VideoFrame` (image + pts_ms) and `AudioChunk` (pcm + start_ms/end_ms).
- `SceneChangeDetector`: Inputs frame sequence, outputs `SceneChangeEvent`/`SceneSegment`.
- `ObjectDetector` (PyTorch): Inputs frame, outputs object/person bounding boxes and classes.
- `FacePipeline` (PyTorch): Face detection → Alignment → Embedding → Comparison with `CharacterStore`.
- `DialogueDetector`: Inputs audio chunk, outputs dialogue interval or real-time state.
- `NarrationGenerator` (TensorFlow): Inputs structured `VisualFacts` and `DialogueState`, outputs `NarrationCandidate`.
- `Synchronizer`: Merges event streams, ensures time consistency, and enforces "No narration during dialogue" rule.
- `OutputSink`: Outputs `NarrationSegment` to JSONL/Callback/Message Queue; Phase 2 TTS will be another sink.

## 4. Performance & Real-time Strategy (Phase 1 Suggestions)

- Decoding Layer: Prioritize hardware decoding (if available), output BGR/RGB frames.
- Sampling Layer: Adopt "Keyframe + Low-frequency refresh" strategy for vision models; Dialogue detection uses continuous audio for stability.
- Parallel Layer: Decoding, Vision Inference, Audio VAD, NLG on separate threads/processes, using bounded queues to avoid accumulation.
- Latency Control: Unified `max_end_to_end_latency_ms` and backpressure strategy.

## 5. Time Synchronization

- Use Video PTS or Audio Clock as the master clock (must fix one), all output events map to this timeline.
- Intervals generated by dialogue detection are used to gate narration output (Allowing summary narration after dialogue ends is an optional feature).
